## Tech Debt (2 items, non-blocking)

1. **Request.halted/violation unused by pipeline** (Phase 13) — The `halted` and `violation` fields on `Request` are defined but never written by `Pipeline.execute_policies/2`. The pipeline returns violations via the `{:error, violation}` tuple instead. These fields exist for future consumer use (e.g., middleware that stamps halted state).

2. **Schema defaults omitted** (Phase 17) — `@guardrails_schema` omits NimbleOptions `default:` values. No impact because `JailbreakDetection` has its own module-level defaults (`@default_threshold 0.7`, `@default_scope :last_message`, `@default_detector Default`).
